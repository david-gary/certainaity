# Changelog

All notable changes to Certainaity are documented here.
Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).
This project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.1.0] — 2026-06-12

### Added

#### Web UI

- React + Vite + Tailwind SPA in `frontend/`.
- Drag-and-drop image upload with format/size validation and inline preview.
- Real-time analysis progress display showing all five pipeline stages.
- Heatmap overlay: bounding boxes drawn on a canvas layer using type-specific colours (red = splicing, yellow = copy-move, purple = AI inpainting, orange = removal).
- Report view: verdict panel, chain-of-custody metadata table, detected-region table with evidence strings, PDF download link.
- Bearer token input persisted across uploads within a session.
- Vite dev server proxies `/v1` to `localhost:8000` to avoid CORS in development.
- `docker/Dockerfile.frontend` — two-stage build (Node 20 builder → nginx:1.25-alpine), SPA fallback routing.
- `frontend` service added to `docker-compose.yml` on port 3000.
- `frontend` CI job: Node 20 setup, `npm install`, typecheck, lint, build.

#### GAN Detection

- `GANDetector` model (`src/certainaity/models/gandec.py`): CLIP ViT-L/14 backbone with global image-level classification head and local 16×16 patch map head.
- Detects images generated by StyleGAN2/3, DALL-E 2/3, Midjourney v5/v6, Stable Diffusion.
- Registered in `Ensemble` with default weight 0.0 (backward-compatible); enable by loading optimized weights once `gandec_v1.pt` is available.
- `ModelName.GAN_DETECTOR` enum variant added.

#### Kubernetes

- `k8s/` directory with full production manifests: namespace, ConfigMap, api Deployment + Service, worker Deployment (GPU node selector + toleration), frontend Deployment + Service, HPA (2–10 replicas on CPU/memory), PVCs for output and weights volumes.

#### Documentation

- MkDocs-material site in `docs/` with 8 pages across three sections: Guides (Getting Started, Docker Deployment, Kubernetes, Contributing), Reference (CLI, REST API, Configuration), Architecture (Overview, Pipeline, Models).
- `docs.yml` GitHub Actions workflow deploys to GitHub Pages on pushes to `main` that touch `docs/` or `mkdocs.yml`.

### Changed

- Trivy container scan in CI now uses direct CLI install (`v0.50.4`) rather than the `aquasecurity/trivy-action` marketplace action.

## [1.0.0] — 2026-02-28

### Added

#### CLI

- `certainaity analyze <image>` entrypoint: ingests an image, runs the full pipeline, and prints a JSON summary to stdout.
- `--output-json` and `--output-pdf` flags for writing report files.
- `--job-id` override to control the report's `job_id` field.

#### API (FastAPI + Celery)

- `POST /v1/analyze` — multipart image upload, enqueues a Celery task, returns a `poll_url`.
- `GET /v1/jobs/{job_id}` — returns task state and stage metadata.
- `GET /v1/reports/{job_id}/json` — serves the completed JSON report.
- `GET /v1/reports/{job_id}/pdf` — serves the completed PDF report.
- `GET /v1/health` — liveness probe (unauthenticated).
- JWT RS256 bearer authentication on protected routes.
- Rate limiting via slowapi: configurable per-IP per-minute limit on `POST /v1/analyze`.

#### Observability

- `GET /metrics` — Prometheus scrape endpoint (unauthenticated, excluded from OpenAPI schema).
- `certainaity_http_requests_total` counter with `method`, `endpoint`, `status_code` labels.
- `certainaity_http_request_duration_seconds` histogram (11 buckets, 5 ms–10 s).
- `certainaity_jobs_submitted_total` and `certainaity_jobs_rejected_total` counters.
- Path normalisation collapses `/v1/jobs/<uuid>` to `/v1/jobs/{job_id}` to prevent cardinality explosion.
- Grafana dashboard (`monitoring/grafana/dashboards/certainaity.json`) with throughput, latency p50/p95/p99, and job queue panels.

#### Forensic Pipeline

- Feature extraction: ELA (error level analysis), wavelet-based noise variance, CFA interpolation correlation, DCT block similarity — all run in parallel via `ThreadPoolExecutor`.
- Ensemble inference: weighted fusion of PatchForensic, MantraNet, SPSL, and InpaintingDetector; falls back to handcrafted feature mean when model weights are absent.
- Resilience test: re-compresses the image at configurable JPEG qualities; flags `anti_forensic_warning` if ensemble confidence drops > 25%.
- Chain-of-custody: SHA-256 computed from raw bytes before any decoding; preserved through task execution into report.

#### Reports

- JSON report with all chain-of-custody fields (`job_id`, `sha256`, `analysis_timestamp`, `manipulation_detected`, `overall_confidence`, `regions`, `models_used`).
- PDF report (ReportLab): header, metadata table, verdict, optional heatmap, detected-region table, chain-of-custody statement.

#### Infrastructure

- Multi-stage production Dockerfiles for api and worker (wheel-only runtime layer, non-root UID 10001, no build tools in final image).
- `docker-compose.override.yml` for CPU-only local development (single replicas, ports 8000/6379 exposed, nginx skipped).
- CI: lint (ruff), type-check (mypy), unit tests (≥ 85% coverage), integration tests, model architecture smoke tests, Trivy container security scan.
- `publish.yml`: builds and pushes `certainaity-api` and `certainaity-worker` to GHCR on `v*` tags; creates GitHub Release from CHANGELOG.

### Changed

- `docker/Dockerfile.api` and `docker/Dockerfile.worker` converted to multi-stage builds.
- Worker image no longer bakes in model weights; weights are mounted as a read-only volume.
